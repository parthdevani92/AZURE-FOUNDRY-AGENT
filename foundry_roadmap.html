Learning roadmap

Your Step-by-Step Path Into Microsoft Foundry

You already know Container Apps, Virtual Machines, Static Web Apps, and Key Vault. This roadmap builds Foundry on top of that — in order, one stage at a time.

8 STAGES · ~6-8 WEEKS AT A STEADY PACE · GO AT YOUR OWN SPEED

Name change to know first: Since 1 January 2026, "Azure AI Foundry" is officially called Microsoft Foundry. It is the exact same product, nothing changed except the name — but tutorials, videos, and old docs may still say "Azure AI Foundry." Treat the two names as the same thing.

What you already bring to this

Foundry does not replace these — it plugs straight into them.

Container Apps Virtual Machines Static Web Apps Key Vault
1

See the big picture

2-3 days

Microsoft Foundry is one website where you build, test, and launch AI apps and AI agents. Nothing to install yet — just explore.

  • Project — a folder that holds your models, agents, and settings.
  • Foundry resource — the actual Azure resource behind your project (same idea as the Azure resource behind a Container App).
  • Portal layout — Discover page (browse), Build page (create), Operate page (monitor).
DoGo to ai.azure.com, sign in with your Azure account, and create one free project.
2

Explore the model catalog

2-3 days

Foundry gives you access to 11,000+ AI models — from OpenAI, Microsoft, Meta, DeepSeek, and others — in one searchable catalog.

  • How to pick and deploy a model (make it ready to use).
  • The Playground — a no-code chat screen to test any model instantly.
  • Basic prompt writing — asking clearly so you get a useful answer.
Side note: Claude — the AI you're talking to right now — is also available inside Microsoft Foundry's model catalog.
DoDeploy a small model (e.g. gpt-5-mini) and chat with it in the Playground.
3

Call a model from your code

~1 week

This is where Foundry starts feeling like real development. You send a prompt from actual code and get an answer back.

  • Install the SDK for Python, C#, or JavaScript — pick the language you know.
  • Sign in from the terminal with az login.
  • Copy your project endpoint (web address) and API key from the portal.
🔗 uses what you know: Key Vault
DoWrite a short script that sends one prompt and prints the answer. Store the key in Key Vault, not in the code.
4

Build your first agent

~1 week

An agent is an AI that follows instructions and can use tools — not just answer once and forget. Foundry gives you two ways to build one.

  • Prompt agent — no code. Write instructions, pick a model, done in the portal.
  • Hosted agent — full code. You bring your own code; Foundry runs it inside a container.
🔗 uses what you know: Container Apps
DoStart with a prompt agent. Give it one small job, like "answer questions about our return policy."
5

Add tools, your own data, and memory

1-2 weeks

A plain model does not know your company's data, and forgets everything after each chat. This stage fixes both problems.

  • Tools — connect your agent to 1,400+ outside systems using MCP (Model Context Protocol), an open standard for plugging in tools.
  • Grounding — use Foundry IQ or Azure AI Search so the agent answers from your own documents, with sources shown.
  • Memory — let the agent remember facts across chats instead of starting fresh every time.
DoUpload one document and ask your agent a question only that document can answer.
6

Multiple agents working together

3-4 days · optional

For bigger tasks, split the work across a few small agents instead of one big one.

  • Microsoft Agent Framework — an open-source toolkit for building and connecting agents.
  • A2A (Agent-to-Agent) — a standard way for one agent to call another agent.
Skip this for now if Stages 4 and 5 don't feel comfortable yet — come back to it later.
7

Deploy for real use

ongoing, start ~1 week

Now your existing Azure skills matter the most.

  • Host the agent's backend as a hosted agent (container-based) or your own Container App.
  • Build the chat screen (frontend) using Static Web Apps.
  • Keep every secret in Key Vault, connected through managed identity (a safer way to connect without passwords).
  • Optional: publish the agent straight into Microsoft Teams.
🔗 Container Apps 🔗 Static Web Apps 🔗 Key Vault
8

Secure, watch, and control cost

ongoing

Add these safety layers before real users touch your agent.

  • RBAC (role-based access control) — decide who can do what.
  • Tracing and evaluation — check if your agent gives good, safe answers.
  • Pricing — Foundry itself is free to explore; you only pay for the models and services you actually use.
DoTurn on tracing for your agent and read one full trace end-to-end.

🔨 A first project that touches everything above

Small enough for a weekend, and it uses your whole stack:

  • Frontend: a simple chat page on Static Web Apps
  • Backend/agent: a Foundry hosted agent, or your own Container App calling the Foundry SDK
  • Secrets: Key Vault + managed identity
  • Data: one PDF or FAQ document grounded through Foundry IQ
